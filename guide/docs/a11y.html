기본정보

  • 2016년 10월, TTAK표준에서 KS국가표준으로 상향 채택
  • 안드로이드 - TalkBack / iOS - VoiceOver
지침1. 대체 텍스트

텍스트가 아닌 콘텐츠는 대체 가능한 텍스트와 함께 제공되어야 한다.

  • 유의사항
    • img alt="이미지 텍스트와 동일한 내용" 으로 입력한다. (의미 없는 이미지는 alt="" 제공)
    • 이미지맵(map, area), longdesc 요소는 스크린리더에서 인식 불가
    • 새창 링크는 반드시 target="_blank" 와 title="새창" 또는 hidden 텍스트를 사용한다.
    • iframe에는 반드시 title=""을 작성한다. (빈 iframe일 경우 title="빈 프레임" 제공)
    • 터치 기반 디바이스이기때문에 대체콘텐츠의 IR 기법 사용에 주의
  • IR 기법
    • text-indent, position 등 대체 콘텐츠를 화면 밖으로 빼거나 z-index로 화면 아래에 깔아놓는 방식은 사용하면 안됨(초점 위치 부적절, 터치 불가능)
    • display:none과 visibility:hidden은 PC 스크린리더와 마찬가지로 인식 불가능
    • font-size:0 으로 숨기면 초점이 인식되지 않으며 터치 불가능
지침2. 자막, 수화 등의 제공

영상이나 음성 콘텐츠에는 동등한 내용의 자막, 원고 또는 수화가제 공되어야 한다.

  • 유의사항
    • 자막(닫힌 자막, Closed Caption), 수화는 반드시 영상과 동기화
    • 원고는 가능하면 영상과 함께 한 화면에 제공
지침3. 색에 무관한 인식

화면에 표시되는 모든 정보는 색에 관계없이 인식될 수 있어야 한다.

  • 유의사항
    • 색상 이외의 레이블, 텍스트, 특수기호, 패턴 등으로도 동등한 내용을 전달할 수 있게 설계
    • 데이터 테이블과 같은 대체 콘텐츠 제공
지침4. 명도 대비

화면에 표시되는 모든 사용자 인터페이스 컴포넌트와 텍스트는 전경색과 배경색이 구분될 수 있도록 제공되어야 한다.
ColourContrast Analyser : https://developer.paciellogroup.com/resources/contrastanalyser/

  • 유의사항
    • 4.5:1 이상의 명도 대비
    • 운영체제 등의 확대 기능을 이용할 수 있는 경우에는 3:1 까지 허용
지침5. 명확한 지시사항

지시사항은 모양, 크기, 위치, 방향, 색, 소리등에 관계없이 인식될 수 있어야 한다.

지침6. 알림기능

알림 정보는 화면표시, 소리, 진동 등 다양한 방법으로 제공되어야 한다.

  • 유의사항
    • 보안키패드 입력시 시각, 청각, 촉각을 활용한 알림 제공 가능
지침7. 초점

의미나 기능을 갖는 모든 사용자 인터페이스 컴포넌트에는 초점(focus)이 적용되고, 초점은 논리적인 순서로 이동되어야 한다.

  • 유의사항
    • 논리적인 순서나 문맥이 적절하게 제공되어야 한다.
    • 화면에 보이는 콘텐츠에만 접근해야 하며 논리적 의미가 없는 객체에는 접근하지 않도록 제공해야 한다.
    • 표시되는 초점의 영역은 콘텐츠의 위치와 크기가 맞도록 제공해야 한다.
  • 제공방법
    • 숨겨진 영역에 초점이 접근되지 않도록 aria-hidden="true" 속성 추가
      <div class="slide"> <div class="banner" aria-hidden="false">1번배너</div> <div class="banner" aria-hidden="true">2번배너</div> </div>
    • 팝업 콘텐츠의 가장 처음에 위치한 최하위 자식요소에 tabindex 속성 제공
      <div class="wrap" aria-hidden="true">본문내용…</div> <div class="popup"> <h3>SK주유권(5천원/1만원권)</h3> <button>닫기</button> <div class="popup_content">팝업콘텐츠…</div> </div>
    • 팝업 콘텐츠로 초점이 자동으로 이동되도록 스크립트 추가
지침8. 누르기 동작 지원

터치(touch) 기반 모바일 기기의 모든 컨트롤은 누르기 동작으로 제어할 수 있어야 한다.

  • 유의사항
    • 두 손가락을 벌리거나 좁혀서 사용하는 지도의 ‘확대/축소’ 기능을 누르기 동작으로도 사용할 수 있게 동일한 기능의 버튼을 제공
    • 슬라이더는 +, - 버튼 등으로도 값을 증감하거나, 입력 서식(Text Field)을 통해 값을 직접 입력
지침9. 응답시간 조절

시간 제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.

  • 유의사항
    • 세션 만료 자동 로그아웃 전에 반드시 로그인 연장 제공
    • 휴대폰번호 본인인증 입력 시간 제한, 반드시 시간 연장 제공
    • "인증번호 재전송" 기능은 SMS로 문자가 재전송되어 제한시간이 3분으로 초기화되므로 제한시간을 연장하거나 제어할 수 있는 수단으로 볼 수 없음
지침10. 정지기능 제공

자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.

  • 유의사항
    • 자동으로 변경되는 콘텐츠에는 정지기능을 반드시 제공해야 함(onfocus 이벤트로 제공하는 것은 지양)
지침11. 컨트롤의크기와간격

컨트롤은 충분한 크기와 간격으로 제공되어야 한다.

  • 유의사항
    • MACAG 2.0은 중심간 간격에 대한 정확한 기준을 제시하고 있지 않음(MAAG 1.0에서는 13mm를 권장)
    • 가로·세로의 크기를 각 9mm 이상으로 제공하는 것을 권장(KWCAG 2.1에서는 대각선 6mm 이상)
    • 링크 텍스트는 예외
지침12. 입력 도움

입력서식 이용 시, 입력 오류를 방지하거나 정정할 수 있는 방법을 제공해야 한다.

  • 유의사항
    • input 등 폼 요소는 반드시 label과 연결한다. (label을 연결할 수 없을 경우 title="" 사용)
    • 대체 정보를 placeholder만으로 인식할 수 있게 구현하면 오류
지침13. 사용자 인터페이스의 일관성

사용자 인터페이스 컴포넌트들은 일관성 있게 배치되어야 한다.

  • 유의사항
    • 스크린리더를 켠 상태에서의 사용자 인터페이스 일관성도 확인
    • 유사한 기능을 가지고 있는 컨트롤(링크, 입력서식, 버튼등)들은 모두 동일한 방식으로 동작하고 기능하도록 제공
지침14. 깜박거림의 사용 제한

깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다.

지침15. 자동재생 금지

자동으로 재생되는 배경음을 사용하지 않아야 한다.

  • 유의사항
    • 동영상의 경우에는 음소거 상태에서 재생이 되도록 제공하는 것도 좋은 방법
지침16. 예측 가능성

사용자가 의도하지 않는 화면 전환이나 이벤트등이 실행되는 경우 사용자가 이해할 수 있는 방법으로 제공되어야 한다.

  • 유의사항
    • 외부 브라우저로 연결되는 링크나 전화번호 링크는 앱이 전환됨을 사전에 안내하기 위해 실행시 팝업을 통해 공지하거나 부가설명으로 안내 필요
지침17. 폰트 관련 기능의 활용

텍스트 콘텐츠는 운영체제에서 제공하는 폰트 관련 기능을 활용할 수 있는 방법을 제공해야 한다.

  • 유의사항
    • 시스템 환경에 설정한 폰트의 형태(확대/축소, 기울임, 굵기 등)를 애들리케이션에 그대로 상속하거나 조절할 수 있는 기능을 제공
지침18. 보조기술과의 호환성

사용자 인터페이스 컴포넌트는 보조기술을 이용하여 사용할 수 있도록 해야 한다.

  • 제공방법(WAI-ARIA)
    역할정보(role) 상태정보(state)
    button 버튼 aria-hidden 숨김 상태
    img 이미지 aria-expanded 확장 상태
    radio 라디오 버튼 aria-checked 선택 상태(radio, check)
    checkbox 체크박스 aria-selected 선택 상태(tab)
    tablist | tab | tabpanel 탭 콘텐츠 aria-pressed 눌림 상태(toggle button)
