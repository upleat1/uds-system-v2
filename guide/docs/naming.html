네이밍 규칙
공통 규칙
  • naming은 영문 소문자, 숫자, 케밥 표기법( - )으로 작성한다.
  • 단어조합시, [형태+의미+숫자+상태]의 형태로 작성한다.
  • 숫자가 없으면 '1'이라는 숫자가 생략된 것으로 간주한다.
  • 약속어를 기본으로 작성하나, 없으면 종류와 특성을 나타내도록 naming한다.
종류 구분 번호 예시
ID Selector 카멜 표기법
(camel-case)		 한 자리수
(생략가능, 2, 3)		 errorMesage, controlCenter, controlCenter2
Class Selector 케밥 표기법
(kebab-case)		 한 자리수
(생략가능, 2, 3)		 tab-page, tbl-type1, tbl-type2
Image 케밥 표기법
(kebab-case)		 두 자리수
(01, 02, 03)		 btn-cancle01-off.gif
파일 및 폴더 케밥 표기법
(kebab-case)		 두 자리수
(01, 02, 03)		 cyber-center
ID 선택자
  • id는 카멜표기법을 사용한다. 예) tabType1
  • 최상위 레이아웃, 폼 요소, 책갈피를 제외하고 id를 사용하지 않는다.
  • 동일한 이름의 id는 문서에서 한번만 사용이 가능하다.
CLASS 선택자
  • class는 케밥표기법을 사용한다. 예) tab-type1
  • 다중의 클래스 지정을 허용하며 최대 3개 이상 사용하지 않도록 주의한다.
이미지 Naming
  • 네이밍 공통 규칙에 따른다.

파일 Naming
  • HTML파일은 화면ID로 작업한다. 예) 화면ID.html
